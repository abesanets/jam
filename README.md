# J.A.M. — Jewelry Atelier Manager

АРМ (автоматизированное рабочее место) ювелирной мастерской. Консольное приложение для Windows на C++14.

---

## Компиляция

```bash
g++ -o jam.exe jam/main.cpp -std=c++14 -finput-charset=UTF-8
```

Требуется MinGW/GCC. Запуск: `jam.exe` из корня проекта (рядом с `users.txt` и `orders.txt`).

---

## Структура проекта

```
├── jam/
│   ├── main.cpp          — точка входа, все экраны и логика навигации
│   ├── models.h          — структуры данных (User, Order)
│   ├── order_manager.h   — CRUD заказов, хранение в orders.txt
│   ├── user_manager.h    — регистрация/авторизация, хранение в users.txt
│   └── ui_manager.h      — отрисовка UI, навигация, цвета
├── orders.txt            — база заказов (открытый текст, pipe-разделитель)
├── users.txt             — база пользователей (XOR+hex шифрование)
└── jam.exe               — скомпилированный бинарник
```

---

## Режимы доступа

### Авторизованный пользователь
Полный доступ к своим заказам: создание, редактирование, смена статуса, удаление.
Просмотр чужих заказов — только чтение.

### Гостевой режим
Доступ без регистрации. Только просмотр заказов, статистика и поиск. Никаких изменений.

---

## Навигация

Везде работают:
- `↑` / `↓` — перемещение по меню и таблицам (с зацикливанием)
- `Enter` — выбор / открыть заказ
- `ESC` — назад / отмена
- Цифры `1`–`9` — быстрый выбор пункта меню

Поддерживается **русская раскладка** — горячие клавиши фильтров работают независимо от языка ввода.

## Экраны

### Стартовый экран
- `[1]` Вход в систему
- `[2]` Регистрация
- `[3]` Войти как гость

### Главное меню (после входа)
- `[1]` Добавить заказ
- `[2]` Управление заказами
- `[3]` Статистика
- `[4]` Поиск

### Управление заказами
Таблица всех заказов с фильтрами и сортировкой. Навигация стрелками, Enter открывает заказ.

| Клавиша | Действие |
|---------|----------|
| `M` | Переключить: мои / все заказы |
| `N` | Фильтр: Новый |
| `W` | Фильтр: В работе |
| `G` | Фильтр: Готов |
| `V` | Показать выданные |
| `D` | Сортировка по дедлайну (цикл: нет → ближе → дальше) |
| `R` | Сбросить все фильтры |

Просроченные заказы подсвечиваются красным.

### Карточка заказа
Детальный просмотр. Если заказ принадлежит текущему пользователю — доступны:
- Изменить статус (выбор из списка стрелками)
- Редактировать поля (пустое поле = не менять)
- Удалить заказ (с подтверждением)

Чужой заказ — только просмотр.

### Поиск
Три режима:
- **По мастеру** — выбор из списка стрелками, показывает все заказы выбранного мастера
- **По ID** — прямой переход к заказу по номеру
- **По тексту** — поиск по ФИО клиента, телефону, описанию и мастеру (регистронезависимый, поддержка кириллицы)

Результаты отображаются в таблице с навигацией стрелками.

### Статистика
- Количество заказов по каждому статусу
- Количество просроченных
- Суммы по всем / активным / выданным заказам

## Хранение данных

### orders.txt
Каждая строка — один заказ, поля разделены `|`:
```
id|ФИО клиента|телефон|описание|цена|статус|дата приёма|дедлайн|мастер
```
Пример:
```
3|Сидорова Елена Павловна|+7-926-555-11-22|Серьги золото 750|28900|В работе|20.03.2026|10.04.2026|Александрович
```

### users.txt
Каждая строка — один пользователь. Все три поля зашифрованы XOR+hex:
```
hex(login)|hex(password)|hex(fullName)
```
Шифрование: XOR каждого байта с циклическим ключом `J4M_s3cr3t_k3y!`, результат кодируется в hex.

## Архитектура кода

### models.h
Структуры данных без логики:
- `User` — логин, пароль, ФИО
- `Order` — все поля заказа
- `ORDER_STATUSES[]` — массив допустимых статусов

### order_manager.h — класс `OrderManager`
Загружает заказы из файла при старте, держит в памяти, сохраняет при каждом изменении.

| Метод | Описание |
|-------|----------|
| `addOrder(...)` | Создать заказ, вернуть ID |
| `getAllOrders()` | Все заказы |
| `findById(id)` | Указатель на заказ по ID |
| `updateStatus(id, status)` | Сменить статус |
| `updateOrder(id, ...)` | Обновить поля |
| `deleteOrder(id)` | Удалить |
| `getFiltered(status, overdue, sort)` | Фильтрация и сортировка |

### user_manager.h — класс `UserManager`
Аналогично: загрузка при старте, шифрование при сохранении.

| Метод | Описание |
|-------|----------|
| `registerUser(login, pass, name)` | Регистрация, проверка уникальности логина |
| `loginUser(login, pass)` | Проверка пары логин/пароль |
| `getFullName(login)` | Получить ФИО по логину |

### ui_manager.h — класс `UIManager`
Все методы статические. Работает напрямую с WinAPI (`SetConsoleCursorPosition`, `SetConsoleTextAttribute`).

Ключевые методы:

| Метод | Описание |
|-------|----------|
| `selectMenu(items, row)` | Меню со стрелками, номерами и Enter |
| `selectFromTable(orders, row, sel, extraKeys)` | Таблица с навигацией, возвращает индекс или код доп. клавиши |
| `normalizeKey(ch)` | UTF-8 кириллица → латиница (QWERTY-маппинг) |
| `inputStringESC(row, label, result)` | Ввод строки с поддержкой Backspace и ESC |
| `drawOrderDetail(order, row)` | Детальная карточка заказа |
| `printCentered(row, text, color)` | Центрированный вывод с цветом |

### Цветовая схема

| Константа | Цвет | Использование |
|-----------|------|---------------|
| `LOGO` | Жёлтый | Заголовки, рамки |
| `HIGHLIGHT` | Жёлтый яркий | Выделенный пункт меню / строка таблицы |
| `MENU` | Голубой | Пункты меню, подсказки |
| `SUCCESS` | Зелёный | Сообщения об успехе |
| `ERROR_CLR` | Красный | Ошибки |
| `OVERDUE` | Красный | Просроченные заказы |
| `DIM` | Серый | Вспомогательный текст |
| `DEFAULT` | Белый | Обычный текст |

| Функция | Описание |
|---------|----------|
| `screenStart` | Стартовый экран |

| `screenRegister` | Форма регистрации |

| `screenOrders` | Управление заказами с фильтрами |
| `screenManageOrder` | Карточка заказа с действиями |
| `screenStats` | Статистика |
| `screenSearch` | Поиск заказов |
| `screenAddOrder` | Форма добавления заказа |
